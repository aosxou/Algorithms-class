## 자료구조 기본 개념

### 1. 선형 구조 vs 비선형 구조

**선형 구조**: 데이터가 일렬로 나열된 형태
- 스택(Stack)
- 큐(Queue)
- 연결 리스트(Linked List)

**비선형 구조**: 데이터가 계층적 또는 네트워크 형태
- 트리(Tree)
- 그래프(Graph)

> **팁**: 선형은 "줄 서기", 비선형은 "가족 관계도"로 생각하면 쉬워요!

## 데이터 표현

### 2. 비트 계산 기본

**필요한 비트 수 계산**: 2^n ≥ 표현할 값의 개수

**예제 02**: 19개의 정보를 구별하려면?
- 2^4 = 16 (부족)
- 2^5 = 32 (충분!)
- **답**: 5비트 필요

**예제 03**: 0부터 63까지 표현하려면?
- 64개의 값 = 2^6
- **답**: 6비트 필요

### 3. 존 형식과 팩 형식

**팩 형식**: 10진수를 압축해서 저장
- 한 바이트에 2자리 숫자 저장
- 마지막 4비트는 부호 (C = +, D = -)

**예시**:
```
+516 → 팩 형식: 51 6C
-516 → 팩 형식: 51 6D
```

**예제 04 풀이**:
```
A: 00 04 09 5C → +4095
B: 00 03 84 0D → -3840
A + B = +4095 + (-3840) = +255
255를 팩 형식으로: 00 00 25 5C
```

## 수의 표현 방식

### 4. 부호와 절댓값 방식

- 최상위 비트: 부호 (0 = +, 1 = -)
- 나머지 비트: 절댓값

**예시 (8비트)**:
```
+62 = 0011 1110
-62 = 1011 1110
```

### 5. 1의 보수

음수 표현: 모든 비트를 반전

**예시**:
```
+62 = 0011 1110
-62 = 1100 0001 (모든 비트 반전)
```

**단점**: 0이 두 개 (+0, -0)

### 6. 2의 보수 ⭐

음수 표현: 1의 보수 + 1

**예시**:
```
+62 = 0011 1110
-62 = 1100 0010 (1의 보수 + 1)
```

**장점**:
- 0이 하나만 존재
- 음수 최댓값이 양수보다 1 크다
- 표현 범위: -2^(n-1) ~ 2^(n-1) - 1

**8비트 범위**: -128 ~ +127

**예제 14 풀이**:
```
10111100₂ (2의 보수)
→ 최상위 비트가 1이므로 음수
→ 다시 2의 보수 취하면: 01000100 = 68
→ 답: -68
```

### 7. 오버플로 판단

**규칙**: 같은 부호끼리 더했는데 부호가 바뀌면 오버플로!

**예제 15**:
- 6비트 범위: -32 ~ +31
- ② 010010(+18) + 001111(+15) = 100001(-33) → 오버플로!

## 부동소수점 표현

### IEEE 754 표준 (32비트)
```
[부호(1)] [지수부(8)] [가수부(23)]
```

**단계**:
1. 정규화: 1.xxxxx × 2^n 형태로 변환
2. 바이어스 적용: 지수 + 127
3. 가수는 소수점 이하만 저장

**예제 16 풀이**:
```
0.01101₂ = 1.101 × 2^(-2)

지수: -2 + 127 = 125 = 01111101₂
답: ② 01111101₂
```

## 알고리즘 기초

### 8. 알고리즘의 5가지 조건

1. **입력**: 0개 이상의 입력
2. **출력**: 1개 이상의 출력
3. **명확성**: 각 명령어가 모호하지 않음
4. **유한성**: 반드시 종료
5. **효과성**: 실행 가능해야 함

> **주의**: 특정 프로그래밍 언어로 작성될 필요는 없어요!

### 9. BCD 코드

**BCD (Binary Coded Decimal)**: 10진수 한 자리를 4비트로 표현

**장점**: 10진수 입출력이 쉬움

**단점**: 저장 공간 낭비 (4비트로 0~9만 사용, 10~15는 미사용)

**예제 20**:
- 4비트로 표현:
  - BCD: 0~9 (10개)
  - 2진화 16진수: 0~15 (16개)
- 차이: 16 - 10 = **6개 더 표현 가능**

## 시간 복잡도

### 10. Big-O 표기법

**성능 순서** (좋음 → 나쁨):
```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2^n)
```

### 11. 복잡도 계산 규칙

**더하기**: 큰 것만 남김
```
O(n) + O(n) = O(n)
O(n²) + O(n) = O(n²)
```

**곱하기**: 그대로 곱함
```
O(log n) · O(n²) = O(n² log n)  ⚠️ O(n²)이 아님!
```

**예제 25**: ④번 오답
```
O(log n) · O(n²) = O(log n · n²) = O(n² log n)
```

### 12. 반복문으로 복잡도 파악하기

**단일 for문**: O(n)
```javascript
for (i = 0; i < n; i++) {
    // O(n)
}
```

**이중 for문**: O(n²)
```javascript
for (i = 0; i < n; i++) {
    for (j = 0; j < n; j++) {
        // O(n²)
    }
}
```

**삼중 for문**: O(n³)
```javascript
for (i = 0; i < n; i++) {
    for (j = 0; j < n; j++) {
        for (k = 0; k < n; k++) {
            // O(n³)
        }
    }
}
```

**예제 29**: 삼중 중첩 → **O(n³)**

**예제 30**: for문이 n에 대해 한 번만 → **O(n)**

### 13. O(1) - 상수 시간

**의미**: 입력 크기와 무관하게 일정한 시간

**예시**:
```javascript
print(array[0]);  // 항상 같은 시간
x = x + 1;        // 항상 같은 시간
```

### 14. 실전 복잡도 분석

**예제 23**: 알고리즘 성능 비교
```
A: O(n²)
B: O(1)         ← 가장 빠름!
C: O(n log n)
D: O(2^n)       ← 가장 느림!

답: B, C, A, D
```

**복잡도 간소화**:
```
2log n + 3n³ + 9999n² + 15
→ 최고차항만: n³
→ 계수 제거: O(n³)
```

## 핵심 요약

### 2의 보수가 널리 쓰이는 이유
- 0의 표현이 하나 (0000 0000)
- 음수 표현 개수가 하나 더 많음
- 덧셈 회로를 그대로 사용 가능

### 시간 복잡도 비교 요령
1. 최고차항만 남긴다
2. 계수는 무시한다
3. 작은 차수는 제거한다

### 완전수 알고리즘 (예제 26)
- 완전수: 자기 자신을 제외한 약수의 합이 자신과 같은 수
- 예: 6 = 1 + 2 + 3
- 1~200 범위에서 완전수: 6, 28

## 연습 문제

### Q1. 2의 보수로 -25를 8비트로 표현하면?
<details>
<summary>답 보기</summary>
```
+25 = 0001 1001
비트 반전 = 1110 0110
+1 = 1110 0111
답: 11100111₂
```
</details>

### Q2. for (i=0; i<n; i++) for (j=0; j<i; j++) 의 복잡도는?
<details>
<summary>답 보기</summary>
```
외부 루프: n번
내부 루프: 0 + 1 + 2 + ... + (n-1) = n(n-1)/2
→ O(n²)
```
</details>

### Q3. 128개 상태를 표현하려면 몇 비트?
<details>
<summary>답 보기</summary>
```
128 = 2^7
답: 7비트
```
</details>

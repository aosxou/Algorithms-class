## 01. 연결 리스트를 사용하기에 적합한 경우는?

**정답: 3) 자료의 삽입과 삭제가 많은 경우**

**풀이:**
- 연결 리스트의 가장 큰 장점은 삽입과 삭제 시 O(1)의 시간복잡도를 가진다는 점
- 배열은 삽입/삭제 시 요소들을 이동시켜야 하므로 O(n)이 소요됨
- 1), 2), 4)번은 배열이 더 효율적
  - 정렬: 배열이 인덱스 접근이 빨라 유리
  - 역순 처리: 배열이 인덱스로 직접 접근 가능
  - 탐색: 배열이 O(1) 랜덤 접근 가능, 연결 리스트는 O(n)

## 02. 연결 리스트에 대한 설명으로 거리가 먼 것은?

**정답: 2) 노드들이 포인터로 연결되어 있어 탐색이 빠르다.**

**풀이:**
- 연결 리스트는 순차 탐색만 가능하므로 탐색 속도가 **느림** (O(n))
- 배열은 인덱스로 직접 접근 가능하여 O(1)
- 1), 3), 4)번은 모두 연결 리스트의 특징
  - 삽입/삭제가 쉬움: 포인터만 변경하면 됨
  - 추가 공간 필요: 각 노드마다 포인터 저장
  - 중간 연결 끊김: 다음 노드 접근 불가능

## 03. 포인터 tmp가 가리키는 노드는?

**정답: 3) 다**

**풀이:**

코드 분석 (일반적인 연결 리스트 순회 패턴):
```c
tmp = head;           // tmp가 '가'를 가리킴
tmp = tmp->link;      // tmp가 '나'를 가리킴
tmp = tmp->link;      // tmp가 '다'를 가리킴
```
- 보통 이런 문제는 head에서 시작해 link를 두 번 따라가는 패턴
- 가 → 나 → 다 순서로 이동

## 04. 시간 복잡도가 가장 낮은 연산은?

**정답: 1) 포인터가 가리키는 노드의 다음 노드를 리스트에서 삭제**

**풀이:**

| 연산 | 시간복잡도 | 이유 |
|------|-----------|------|
| 1) 다음 노드 삭제 | **O(1)** | 포인터 변경만으로 가능 |
| 2) 리스트 길이 출력 | O(n) | 모든 노드를 순회해야 함 |
| 3) 임의의 노드 앞에 삽입 | O(n) | 해당 노드까지 탐색 필요 |
| 4) 마지막 노드 접근 | O(n) | 끝까지 순회해야 함 |

```c
// O(1) 삭제 코드
temp = ptr->link;          // 삭제할 노드 저장
ptr->link = temp->link;    // 포인터 연결 변경
free(temp);                // 메모리 해제
```

## 05. 자료 B를 삭제한 후 변경된 내용은?

**정답: 1) A의 link -> 2030**

**풀이:**

**삭제 전:**
```
A(2010) -> B(2020) -> C(2030)
```

**삭제 과정:**
```c
A->link = B->link;  // A의 link를 C의 주소(2030)로 변경
```

**삭제 후:**
```
A(2010) -> C(2030)
```

- A의 link가 B를 건너뛰고 C(주소 2030)를 직접 가리킴
- B는 리스트에서 분리됨

## 06. 노드 tmp를 first와 second 사이에 삽입하는 코드는?

**정답: 2) tmp.link = first.link; first.link = &tmp;**

**풀이:**

**삽입 전:**
```
first -> second
```

**올바른 순서:**
```c
tmp.link = first.link;   // 1단계: tmp가 second를 가리키게
first.link = &tmp;       // 2단계: first가 tmp를 가리키게
```

**삽입 후:**
```
first -> tmp -> second
```

**주의:** 순서가 바뀌면 second를 잃어버림!

**각 선택지 분석:**
- 1) A의 link를 first로 바꿔버림 (역방향)
- 2) ✅ 올바른 순서
- 3) second.link를 변경하려 함 (잘못된 접근)
- 4) tmp를 끝에 연결, second와의 연결 끊김

## 07. foobar 함수가 하는 일은?

**정답: 3) 원형으로 연결된 단순 연결 리스트에서 *ptr이 가리키는 노드의 뒤에 node를 삽입, 리스트가 비어 있으면 node로 새로 리스트를 만든다.**

**풀이:**

일반적인 원형 연결 리스트 삽입 코드 패턴:
```c
void foobar(NODE **ptr, NODE *node) {
    if(*ptr == NULL) {           // 빈 리스트
        *ptr = node;
        node->link = node;        // 자기 자신을 가리킴 (원형)
    } else {
        node->link = (*ptr)->link;  // 새 노드가 다음을 가리킴
        (*ptr)->link = node;        // 현재 노드가 새 노드를 가리킴
    }
}
```

**원형 리스트의 특징:**
- 마지막 노드가 첫 노드를 가리킴
- 빈 리스트일 때 노드가 자기 자신을 가리킴 (`node->link = node`)
- NULL이 없음 (계속 순환)

**단순 연결 리스트 vs 이중 연결 리스트:**
- 단순: 한 방향 포인터만 (link)
- 이중: 양방향 포인터 (prev, next)

## 08. 연결 리스트 역순 변환 알고리즘의 빈칸은?

**정답: 3) r = q; q = p -> link;**

**풀이:**

**전체 알고리즘 구조:**
```c
p = start;
q = NULL;
r = NULL;

while(p != NULL) {
    r = q;              // 이전 노드 저장
    q = p;              // 현재 노드 이동
    p = p->link;        // 다음 노드로 전진
    q->link = r;        // 포인터 역방향 연결
}
start = q;              // 새로운 시작점
```

**동작 과정:**
```
초기: A -> B -> C -> NULL

1회전: NULL <- A    B -> C -> NULL
       r    q    p

2회전: NULL <- A <- B    C -> NULL
            r    q    p

3회전: NULL <- A <- B <- C
                 r    q    p(NULL)
```

**빈칸 ㄱ의 역할:**
- `r = q`: 이전 노드 위치 저장
- `q = p->link`: 다음 처리할 노드로 이동 준비

**3개 포인터의 역할:**
- `p`: 현재 처리 중인 노드 (전진용)
- `q`: 포인터를 바꿀 노드
- `r`: 이전 노드 (역방향 연결용)

## 핵심 개념 정리

### 연결 리스트 시간복잡도

| 연산 | 시간복잡도 | 비고 |
|------|-----------|------|
| 삽입 (위치 알 때) | O(1) | 포인터만 변경 |
| 삭제 (위치 알 때) | O(1) | 포인터만 변경 |
| 탐색 | O(n) | 순차 접근만 가능 |
| 삽입 (위치 모를 때) | O(n) | 탐색 + 삽입 |

### 연결 리스트 vs 배열

| 구분 | 연결 리스트 | 배열 |
|------|------------|------|
| 삽입/삭제 | O(1) - 빠름 | O(n) - 느림 |
| 탐색 | O(n) - 느림 | O(1) - 빠름 |
| 메모리 | 추가 공간(포인터) | 연속적 할당 |
| 크기 | 가변적 | 고정적 |

### 연결 리스트 종류

1. **단순 연결 리스트**: 한 방향으로만 연결
2. **이중 연결 리스트**: 양방향 연결 (prev, next)
3. **원형 연결 리스트**: 마지막이 처음을 가리킴

### 연결 리스트 기본 연산

**삽입 (A와 B 사이에 C 삽입):**
```c
C->link = A->link;   // 1. C가 B를 가리킴
A->link = C;         // 2. A가 C를 가리킴
```

**삭제 (A와 C 사이의 B 삭제):**
```c
temp = A->link;      // 1. B 저장
A->link = temp->link; // 2. A가 C를 가리킴
free(temp);          // 3. B 메모리 해제
```

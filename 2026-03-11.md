## 1. 자료구조의 이해

### 개념
- 자료를 **효율적으로 표현·저장·처리**할 수 있도록 정리하는 것
- 컴퓨터가 효율적으로 문제를 처리하려면 최적의 프로그램이 필요 → 자료구조 개념과 활용 능력 필수

### 문제 해결 과정
```
문제 정의 → [처리 방식 결정 → 알고리즘 작성]  → 프로그램 작성 → 번역 및 실행 → 결과
          → [처리 대상 결정 → 자료 정의    ]
```

### 자료구조 분류

| 구조 | 종류 |
|------|------|
| **단순 구조** | 정수, 실수, 문자, 문자열 |
| **선형 구조** | 순차 리스트, 연결 리스트, 스택, 큐, 데크 |
| **비선형 구조** | 트리, 그래프 |
| **파일 구조** | 순차 파일, 색인 파일, 직접 파일 |

- 선형 구조: 자료 간 관계 **1:1**
- 비선형 구조: 자료 간 관계 **1:다, 다:다**

## 2. 자료의 표현

### 2진수 코드 단위

| 단위 | 크기 |
|------|------|
| 비트(Bit) | 1비트 |
| 니블(Nibble) | 4비트 |
| 바이트(Byte) | 8비트 = 2니블 |

- n개의 비트 → **2ⁿ**개의 상태 표현 가능

---

### 수치 자료 표현

#### 10진수 — 존(Zone) 형식
- 10진수 한 자리 = **1바이트**
- 상위 4비트(존 영역): `1111` 고정
- 하위 4비트(수치 영역): 해당 숫자의 2진수
- 부호: 마지막 자리 존 영역에 표시 (`1100` = 양수, `1101` = 음수)

#### 10진수 — 팩(Pack) 형식
- 존 영역 없이 **4비트**만으로 한 자리 표현 (더 효율적)
- 최하위 4비트에 부호 표시 (`1100` = 양수, `1101` = 음수)

#### 2진수 정수 표현 3가지

| 방식 | 특징 |
|------|------|
| **부호 절댓값** | MSB가 부호 비트 (0=양수, 1=음수). +0, -0 둘 다 존재 |
| **1의 보수** | 음수 = 각 비트 반전. +0, -0 둘 다 존재 |
| **2의 보수** | 음수 = 1의 보수 + 1. 컴퓨터 시스템에서 **실제 사용** |

> 세 방식 모두 **양수 표현은 동일**, 음수 표현만 다름

#### 2진수 실수 표현

- **고정 소수점**: 소수점이 최상위 비트 왼쪽에 고정
- **부동 소수점**: `실수 = 소수부 × 밀수^지수` 형태로 표현, 범위가 더 넓음
  - 4바이트: 부호(1) + 지수부(8) + 가수부(23)
  - 8바이트: 부호(1) + 지수부(11) + 가수부(52)

---

### 문자 자료 표현

| 코드 | 비트 수 | 특징 |
|------|---------|------|
| **BCD** | 6비트 | 존(2) + 숫자(4). 대문자·숫자·특수문자 |
| **EBCDIC** | 8비트 | 존(4) + 숫자(4). 대소문자·숫자·특수문자 |
| **ASCII** | 7비트 | 존(3) + 숫자(4). 현재 가장 범용적 |
| **유니코드** | 16비트(2바이트) | 국제 표준(ISO/IEC 10646). 다국어 지원 |

> 역사: BCD → EBCDIC → ASCII → **유니코드** (현재 일반화)

---

### 기타 자료 표현

- **논리 자료**: 참(True)/거짓(False) 표현. 1바이트 사용
- **포인터 자료**: 메모리 주소를 저장하는 자료형
- **문자열 자료**: 여러 문자를 연속 저장

#### 문자열 표현 방법 비교

| 방법 | 메모리 이용률 | 탐색 시간 |
|------|-------------|----------|
| 구분자 사용 | 효율적 | 비효율적 |
| 고정 길이 저장 | 비효율적 | 효율적 |
| 포인터 사용 | 효율적 | 효율적 |

## 3. 자료의 추상화

### 핵심 개념

- **추상화**: "무엇(what)인가?" → 논리적으로 정의
- **구체화**: "어떻게(how) 할 것인가?" → 실제적으로 표현

| 구분 | 자료 | 연산 |
|------|------|------|
| 추상화 | 추상 자료형(ADT) | 알고리즘 정의 |
| 구체화 | 자료형 | 프로그램 구현 |

### 추상 자료형 (ADT, Abstract Data Type)
- 자료와 연산자의 특성을 **논리적으로 추상화**하여 정의한 자료형
- 예) 정수형 ADT: 자료 = `{…, -1, 0, 1, …}`, 연산 = `{+, -, ×, ÷, mod}`

## 4. 알고리즘의 이해

### 정의
> 문제해결 방법을 추상화하여 **단계적 절차를 논리적으로 기술**해 놓은 명세서

### 알고리즘의 5가지 조건

| 조건 | 설명 |
|------|------|
| **입력** (Input) | 외부에서 0개 이상의 입력 제공 가능 |
| **출력** (Output) | 1개 이상의 결과 출력 |
| **명확성** (Definiteness) | 명령어가 명확하게 명세 |
| **유한성** (Finiteness) | 반드시 종료되어야 함 |
| **효과성** (Effectiveness) | 모든 명령이 기본적이고 실행 가능해야 함 |

## 5. 알고리즘의 표현 방법

### 표현 방법 종류
1. 자연어를 이용한 서술적 표현
2. 순서도(Flow chart)를 이용한 도식화
3. 프로그래밍 언어를 이용한 구체화
4. **가상코드(Pseudo-code)** 를 이용한 추상화 ← 주로 사용

### 순서도 기호

| 기호 모양 | 의미 |
|----------|------|
| 타원 | 시작, 종료 |
| 직사각형 | 처리문, 치환문 |
| 평행사변형 | 입력, 출력 |
| 마름모 | 조건문, 판단문 |
| 화살표 | 제어 흐름 |

### 가상코드 주요 제어 구조

#### 조건문
```
if (조건식) then 명령문1;
else 명령문2;
```

#### 다단계 조건문 (중첩 if)
```
if (조건식1) then 명령문1;
else if (조건식2) then 명령문2;
else 명령문3;
```

#### case문
```
case {
    조건식1 : 명령문1;
    조건식2 : 명령문2;
    else    : 명령문n;
}
```

#### 반복문
```
// for문
for (초깃값; 조건식; 증감값) do 명령문;

// while-do문
while (조건식) do 명령문;

// do-while문
do 명령문;
while (조건식);
```

#### 함수문
```
함수 이름(매개변수)
    명령문;
    ...
    return 결괏값;
end
```

## 6. 알고리즘의 성능 분석

### 성능 분석 기준
- 정확성, 명확성, 수행량, 메모리 사용량, **최적성** (가장 중요)

### 공간 복잡도 (Space Complexity)
```
공간 복잡도 = 고정 공간 + 가변 공간
```

### 시간 복잡도 (Time Complexity)
```
시간 복잡도 = 컴파일 시간 + 실행 시간
```
- 컴파일 시간: 거의 고정
- 실행 시간: **명령문의 실행 빈도수**로 계산

### 점근 표기법 3가지

| 표기법 | 읽기 | 의미 |
|--------|------|------|
| **O(f(n))** | Big-Oh | **상한** — 최악의 경우에도 이 시간 내 완료 보장 |
| **Ω(f(n))** | Big-Omega | **하한** — 적어도 이 시간 필요 |
| **θ(f(n))** | Big-Theta | **상한 = 하한** — 정확한 차수 |

### 시간 복잡도 크기 순서
```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ)
```

### 시간 복잡도에 따른 수행 시간 비교 (n=1,000,000 기준)

| 복잡도 | 수행 시간 |
|--------|----------|
| O(n) | 10⁻³초 |
| O(n log n) | 2×10⁻²초 |
| O(n²) | 17분 |
| O(n³) | 32년 |
| O(2ⁿ) | 사실상 불가능 |

> 빅-오 표기법 계산법: 실행 빈도수 함수에서 **가장 영향이 큰 항** 선택 후 계수 생략
> 예) `4n + 2` → `O(n)`
